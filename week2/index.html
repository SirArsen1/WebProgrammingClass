1. Color

CSS uses color values to style text, backgrounds, borders, and more. 

/* Color always mean text color */
color: orange;
/* Background color */
background-color: #ecf0f1;
/* Border color */  
border-color: 1px solid #e74c3c;

This text is colored with color.

This paragraph has a background-color.

This box has a red border.

CSS offers several ways to specify colors. Every color value can also include an alpha channel to control opacity.

Color Names

CSS supports 140 named colors like red, lightblue, or even tomato. For a full list of supported names, see MDN Web Docs.

RGB / RGBA

rgb(r, g, b) uses decimal numbers from 0 to 255. Add a fourth value for alpha opacity: rgba(r, g, b, a) where alpha is a number between 0 (fully transparent) and 1 (fully opaque).

HEX

A six-digit hexadecimal notation: #rrggbb. The first two digits stand for red, the next two for green, the last two for blue. Values range from 00 (none) to ff (full). So #ffffff is white, #000000 is black, and #ff0000 is red. You can use the shorter version #rgb when each pair has identical digits, e.g. #333 equals #333333.

HSL / HSLA

hsl(h, s%, l%) stands for hue (0–360 on the color wheel), saturation (0%–100%), and lightness (0%–100%). Like RGB, an alpha channel can be added: hsla(h, s%, l%, a).

It is easier to create a color palette with HSL, while RGB is more common for precise color matching. Named colors are convenient for quick styling but less flexible.

/* Named color */
color: tomato;

/* HEX */
color: #ff6600;
color: #f60;        /* shorthand */

/* RGB / RGBA */
color: rgb(255, 102, 0);
color: rgba(255, 102, 0, 0.6);

/* HSL / HSLA */
color: hsl(24, 100%, 50%);
color: hsla(24, 100%, 50%, 0.6);

Opacity

The opacity property sets the transparency for an entire element (including its children), whereas rgba / hsla only affect a single background or color.

/* Makes the whole box (and its text) semi-transparent */
div { opacity: 0.5; }

2. CSS Selectors

Selectors define which elements a CSS rule applies to. Below are the most common three cases:

/* Type selector — targets all <p> elements */
p { color: #2c3e50; }

/* Class selector — targets elements with class="highlight" */
.highlight { background: #ffeaa7; }

/* ID selector — targets the element with id="special" */
#special { border: 2px solid #e74c3c; }

Child Selector (parent > child)

Targets elements that are direct children of a parent.

<style>
    /* Only <a> that is a direct child of .child-demo */
    .child-demo > a { color: #e74c3c; font-weight: bold; }
</style>

Attribute Selectors

Target elements based on the presence or value of an attribute. One can use various operators to match exact values, partial values, or even the presence of an attribute. See MDN Web Docs for all the possibilities.

/* Elements with data-type "primary" */
[data-type="primary"] { 
    background: #3498db; 
}

/* Links starting with https */
a[href^="https"]{ 
    color:purple;
}

/* Classes containing "example" -> class is a special attribute*/
span[class*="example"] {
    color: #ff0000; 
}

Pseudo-classes

Another way to specify conditions is with pseudo-classes, which target elements in a certain state. For example, :first-child targets an element that is the first child of its parent,:last-child targets the last child, and :nth-child(n) targets the nth child. There are also pseudo-classes for user interaction like :hover, :focus, and :active. We will talk more about them later.

<style>
    .pseudo-demo :first-child { color: red; }
    .pseudo-demo :last-child { color: green; }
    .pseudo-demo :nth-child(2) { color: orange; }
</style>
<div class="pseudo-demo">
    <p>First child (red)</p>
    <p>Second child (orange)</p>
    <p>Third child (green)</p>
</div>

Selectors can be combined to target elements with multiple conditions. For example: 

/* Target the first li that is a child of ul with id combined */
ul#combined li:first-child { 
    color: teal;

3. CSS Rules Cascade

When multiple CSS rules target the same element, the browser follows a cascade to decide which one wins. Three principles govern this:

  • Specificity: A more specific selector (e.g. #id) beats a less specific one (e.g. .class), which beats a type selector.
  • Last Rule: If two selectors have the same specificity, the one declared last in the stylesheet wins.
  • !important: A value marked !important overrides all normal declarations — use it sparingly.
<style>
.cascade-box p {
    color: grey; 
}

/* Both rules target the same class — the latter wins */
.cascade-box .text {
    color: green;  /* this one is ignored */
} 
.cascade-box .text { 
    color: lightblue; /* this one is applied */
}   

/* #id wins over .class */
.cascade-box #important-text { 
    color: orange ; 
}
</style>
<div class="cascade-box">
    <p>This is a normal paragraph without class</p>
    <p class="text">This text uses the <em>last</em> rule → lightblue.</p>
    <p class="text" id="important-text" >This one has important-text as id. — orange.</p>
</div>

This is a normal paragraph without class

This text uses the last rule → lightblue.

This one has important-text as id. — orange.

Specificity hierarchy (strongest to weakest): inline styles → #id → .class → type. The last rule only matters when specificity is equal. !important sits above everything (but two !importants fall back to specificity and source order).

4. Text Styling

CSS provides a lot of properties to control the appearance of text.

Font Family

Use font-family to specify the typeface. Provide fallback options separated by commas. The browser will use the first available font in the list. Mac and Windows have different default fonts, so it's common to include both along with a generic family as the last fallback.

font-family: 'Helvetica', Arial, sans-serif;
font-family: 'Georgia', 'Times New Roman', serif;
font-family: 'Courier New', monospace;

Sans-serif (Helvetica / Arial)

Serif (Georgia)

Monospace (Courier New)

If one wants to use a custom font that may not be available on all devices, they can use the @font-face rule to load a font file from a URL. This allows for consistent typography across different platforms.

@font-face {
    font-family: 'MyCustomFont';
    src: url('https://example.com/fonts/mycustomfont.woff2') format('woff2');
}
body {
    font-family: 'MyCustomFont', sans-serif;
}

This text uses MyCustomFont if available, otherwise falls back to sans-serif.

Font Size

font-size can be set in pixels (px), ems (em), rems (rem), percentages, or keywords (small, medium, large). 

font-size: 16px;
font-size: 1.2em;
font-size: 1.2rem;
font-size: 120%;
font-size: medium;

Font Weight & Style

font-weight controls the thickness of the text. Common values are normal, bold, black or numeric weights from 100 to 900 (where 400 is normal and 700 is bold). font-style controls whether the text is italicized or not, with values like normal, italic, and oblique.

font-weight: bold;        /* also: normal, 100–900 */
font-weight: 700;
font-style: italic;       /* also: normal, oblique */

Text Transform & Decoration

text-transform controls capitalization while not affecting the actual text content, this is very useful when one wants to add text transformation only for the visual styles.
text-decoration adds lines like underline or strikethrough. Note that links are underlined by default, but this can be overridden with text-decoration: none;.

text-transform: uppercase;   /* uppercase, lowercase, capitalize */
text-decoration: underline;  /* underline, overline, line-through, none */

Line Height & Spacing

line-height controls the vertical spacing between lines of text. A unitless value (e.g. 1.5) multiplies the font size, while a length value (e.g. 24px) sets an absolute line height. letter-spacing adjusts the space between characters, and word-spacing adjusts the space between words.

line-height: 1.6;           /* unitless multiplier */
letter-spacing: 2px;        /* space between characters */
word-spacing: 4px;          /* space between words */

5. Boxes

Every element in CSS is a rectangular box. The box model consists of content, padding, border, and margin — in that order, from inside out. The tricky thing is that the width and height you set with CSS only apply to the content area by default, so the actual size of the element will be larger once you add padding and borders. 

/* What's the total width of this element? */
                #demo-box {
                    width: 200px;           /* content width */
                    height: 50px;
                    padding: 10px;          /* space between content and border */
                    border: 2px solid #e74c3c; /* border width, style, and color */
                    margin: 20px;           /* space outside the border */
                    background:#3498db;
                }
This box has margin, border, and padding.

Border

border can be set with individual properties for width, style, and color, or with a shorthand that combines all three. You can also specify borders for individual sides (e.g. border-top, border-right, etc.).

/* Individual properties */
border-width: 2px;
border-style: solid;    /* solid, dashed, dotted, double, ... */
border-color: #e74c3c;

/* Shorthand: width style olor; */
border: 2px dashed #3498db;

/* Individual sides */
border-top: 3px solid #2ecc71;

Padding & Margin

Padding is the space inside the border (between content and border). Margin is the space outside the border (between elements). Similar as with border, you can set padding and margin for all sides at once or for individual sides.


/* all sides → 10px */
padding: 10px;
/* top/bottom → 5px, left/right → 30px */
padding: 5px 30px; 
/* Same as: top right bottom left */
padding: 5px 30px 5px 30px;

/* Individual sides */
padding-top: 1rem;

Overflow

When content is larger than its box, use overflow to control what happens.

overflow: hidden;    /* clips excess content */
overflow: scroll;    /* always shows scrollbars */
overflow: auto;      /* shows scrollbars only when needed */

Display/Visibility

Remember that some HTML elements are block-level by default (e.g. <div>, <p>), while others are inline (e.g. <span>, <a>)? The display attribute controls how an element is displayed. The most common values are block, inline, and inline-block. A block element takes up the full width available and starts on a new line, while an inline element only takes up as much width as necessary and can sit next to other inline elements. An inline-block element is like a block element(padding/margin...) but can flow next to other inline or inline-block elements.

See how a simple navigation menu is created using an unordered list below by changing its default display value;

6. Inheritance

Some CSS properties are inherited from parent to child. Text-related properties like font-family, color, and font-size are typically inherited, while box-model properties like border, padding, and margin are not. 

/* Children inherit font-family and color from .inherit-demo */
.inherit-demo {
    font-family: Georgia, serif;
    color: #2c3e50;
}

/* Border is NOT inherited — you must set it directly */
.inherit-demo a { border: 1px solid #e74c3c; }

This paragraph inherits Georgia and the dark color.

A link inside inherits the color but not the border — the border had to be set explicitly on a.

This nested div also inherits the font; its border is set directly on this element.

You can use the inherit keyword to force inheritance of a normally non-inherited property, and initial to reset to the browser's default value. 

<div style="color:blue;">
  Text inside this div is set to blue
  <p>This paragraph inherits the color from its parent by default → blue.</p>
  <p style="color:initial;">This paragraph is reset to the default color → black.</p>

    div outside
    <div style="border:inherit;">div inside </div>
  </div>
</div>
